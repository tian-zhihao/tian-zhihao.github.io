JAVA注解
java java基础
java基础
发布日期:   2018-10-29
阅读次数:  

说明：接触Spring之后发现JAVA注解的使用非常广泛，但之前学习JAVA基础时，没有重视这部分内容。今天看到Spring AOP基于注解的拦截相关内容，利用了自定义注解。故回过头学习注解相关内容。看了些博客，并自己写了些样例帮助理解。

一、JAVA注解

  1. 注解是JDK5之后才有的新特性
  2. 作用相当于注释，或标记，可以标记在类上的不同位置，修饰成员变量、方法、方法参数或类。（就是因为觉得它是解释类的，最开始才没有加以重视的）
  3. 注解的重要性体现在：javac编译器等可以通过反射来了解各自元素上的标记（注解）依据标记去干相应的事情。比如Spring AOP 基于注解的拦截（advice）。

二、内置注解

JDK1.5之后内部提供的三个注解

​ @Deprecated 意思是“废弃的，过时的”

​ @Override 意思是“重写、覆盖”

​ @SuppressWarnings 意思是“压缩警告”

1.@Deprecated :废弃的

源码：

@Documented
@Retention(RetentionPolicy.RUNTIME)
@Target(value={CONSTRUCTOR, FIELD, LOCAL_VARIABLE, METHOD, PACKAGE, PARAMETER, TYPE})
public @interface Deprecated {
}

应用：

thread中的suspend()、destroy()、resume()方法：

/**
 * Suspends this thread.
 .......
 * @deprecated   This method has been deprecated, as it is
 *   inherently deadlock-prone.  If the target thread holds a lock on the
 *   monitor protecting a critical system resource when it is suspended, no
 *   thread can access this resource until the target thread is resumed. If
 *   the thread that would resume the target thread attempts to lock this
 *   monitor prior to calling <code>resume</code>, deadlock results.  Such
 *   deadlocks typically manifest themselves as "frozen" processes.
 *   For more information, see
 *   <a href="{@docRoot}/../technotes/guides/concurrency/threadPrimitiveDeprecation.html">Why
 *   are Thread.stop, Thread.suspend and Thread.resume Deprecated?</a>.
 */
@Deprecated
public final void suspend() {
    checkAccess();
    suspend0();
}

2.@Override:重写

源码：

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.SOURCE)
public @interface Override {
}

3.@SuppressWarnings :警告抑制

源码：

@Target({TYPE, FIELD, METHOD, PARAMETER, CONSTRUCTOR, LOCAL_VARIABLE})
@Retention(RetentionPolicy.SOURCE)
public @interface SuppressWarnings {
    /**
     * The set of warnings that are to be suppressed by the compiler in the
     * annotated element.  Duplicate names are permitted.  The second and
     * successive occurrences of a name are ignored.  The presence of
     * unrecognized warning names is <i>not</i> an error: Compilers must
     * ignore any warning names they do not recognize.  They are, however,
     * free to emit a warning if an annotation contains an unrecognized
     * warning name.
     *
     * <p> The string {@code "unchecked"} is used to suppress
     * unchecked warnings. Compiler vendors should document the
     * additional warning names they support in conjunction with this
     * annotation type. They are encouraged to cooperate to ensure
     * that the same names work across multiple compilers.
     * @return the set of warnings to be suppressed
     */
    String[] value();
}

理解：用于抑制警告，可用于类、字段、函数、函数入参、构造函数和函数的局部变量。而家建议注解应声明在最接近警告发生的位置。

关键字：

关键字 用途
all to suppress all warnings
boxing to suppress warnings relative to boxing/unboxing operations
cast to suppress warnings relative to cast operations
dep-ann to suppress warnings relative to deprecated annotation
deprecation to suppress warnings relative to deprecation
fallthrough to suppress warnings relative to missing breaks in switch statements
finally to suppress warnings relative to finally block that don’t return
hiding to suppress warnings relative to locals that hide variable
incomplete-switch to suppress warnings relative to missing entries in a switch statement (enum case)
nls to suppress warnings relative to non-nls string literals
null to suppress warnings relative to null analysis
rawtypes to suppress warnings relative to un-specific types when using generics on class params
restriction to suppress warnings relative to usage of discouraged or forbidden references
serial to suppress warnings relative to missing serialVersionUID field for a serializable class
static-access o suppress warnings relative to incorrect static access
synthetic-access to suppress warnings relative to unoptimized access from inner classes
unchecked to suppress warnings relative to unchecked operations
unqualified-field-access to suppress warnings relative to field access unqualified
unused to suppress warnings relative to unused code

举例：

@SuppressWarnings(value={"unchecked", "rawtypes"})
public void addItems(String item){
   List items = new ArrayList();
   items.add(item);
}

三、自定义注解

1.基本语法

定义注解：

 public @interface MyAnnotation{}

使用注解：

@ MyAnnotation 
public class AnnotationUse{

}

2.用于自定义注解的注解

1.@Target：注解的目标对象
表示该注解可以用于什么地方，可能的ElementType参数有：

CONSTRUCTOR：构造器的声明
FIELD：域声明（包括enum实例）
LOCAL_VARIABLE：局部变量声明
METHOD：方法声明
PACKAGE：包声明
PARAMETER：参数声明
TYPE：类、接口（包括注解类型）或enum声明

2.@Retention：生命周期

表示需要在什么级别保存该注解信息。可选的RetentionPolicy参数包括：
SOURCE：注解将被编译器丢弃
CLASS：注解在class文件中可用，但会被VM丢弃
RUNTIME：VM将在运行期间保留注解，因此可以通过反射机制读取注解的信息

3.@Document：将注解包含在Javadoc中
4.@Inherited：允许子类继承父类中的注解

3.带属性的注解

  • 注解可以带属性
  • 当定义的注解中只有一个属性value的时候，在使用此注解时，对其value属性赋值可以不必明确写上value而是直接使用属性值即可
  • 当定义的注解中有value属性，同时也包含有其他属性时候，那么在对属性赋值时候，必须明确的以name = value的形式赋值。如果注解中存在数组属性，那么在对其赋值的时候，如果是单个值，可不使用“{}”的形式，如果是多个值，必须使用“{}”。
  • 也可以对属性值直接赋予默认值，使用default关键字，那么在赋值的时候，可以不指定值，那么使用的就是默认值，同时也可以对于赋予新的值。

四、自定义注解的使用

基于注解是一种趋势，可以简化代码，Spring2.5以上都是基于注解的，Hibernate3.x以后也是基于注解的，现在的Struts2有一部分也是基于注解的。注解通常配合反射机制一起使用。

例子1：通过反射查看自定义注解信息

  1. 写自定义注解

    @Target(value = {ElementType.FIELD,ElementType.CONSTRUCTOR,ElementType.METHOD,ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
@interface MyAnnotation{
    String name() default "我的注解";
}

  2. 写标记该注解的类

    @MyAnnotation(name="这是一个类注解")
class User {
    @MyAnnotation(name="这是一个域注解")
    String name;
    @MyAnnotation(name="这是一个构造器注解")
    public User(String name) {
        this.name = name;
    }
    @MyAnnotation(name="这是一个方法注解")
    public void setName(String name) {
        this.name = name;
    }
}

  3. 利用反射查看自定义注解的信息

    public class Main{
    public static void main(String [] args) throws Exception{
       Class clazz=User.class;
       System.out.println("User类上的注解："+clazz.getAnnotation(MyAnnotation.class));
       System.out.println("User构造器上的注解："+clazz.getConstructor(new Class[]{String.class}).getAnnotation(MyAnnotation.class));
       System.out.println("User域上的注解："+clazz.getDeclaredField("name").getAnnotation(MyAnnotation.class));
        System.out.println("User方法上的注解："+clazz.getDeclaredMethod("setName",new Class[]{String.class}).getAnnotation(MyAnnotation.class));
    }

  4. 执行结果：

    User类上的注解：@MyAnnotation(name=这是一个类注解)
    User构造器上的注解：@MyAnnotation(name=这是一个构造器注解)
    User域上的注解：@MyAnnotation(name=这是一个域注解)
    User方法上的注解：@MyAnnotation(name=这是一个方法注解)

例子2：Spring AOP 中基于自定义注解做拦截

  1. 写拦截规则注解：

    import java.lang.annotation.Documented;
import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface Action {
    String name();
}

  2. 写使用注解的被拦截类：

    import org.springframework.stereotype.Service;

@Service
public class DemoAnnotationService {
   @Action(name="注解式拦截的add操作")
    public void add(){} 

}

  3. 编写切面：

    import java.lang.reflect.Method;

import org.aspectj.lang.JoinPoint;
import org.aspectj.lang.annotation.After;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.aspectj.lang.annotation.Pointcut;
import org.aspectj.lang.reflect.MethodSignature;
import org.springframework.stereotype.Component;

@Aspect 
@Component 
public class LogAspect {

   @Pointcut("@annotation(com.wisely.highlight_spring4.ch1.aop.Action)") //1
   public void annotationPointCut(){};

     @After("annotationPointCut()") 
       public void after(JoinPoint joinPoint) {
           MethodSignature signature = (MethodSignature) joinPoint.getSignature();//2
           Method method = signature.getMethod();//3
           Action action = method.getAnnotation(Action.class);//4
           System.out.println("注解式拦截 " + action.name()); 
       } 
}

    解释：

    • 1：声明基于注解的切面
    • 2：从连接点取签名
    • 3：从签名取方法
    • 4：从方法取注解
